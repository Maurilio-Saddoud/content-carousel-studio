# Most teams say they want agents.
# They usually mean four different things.

---

# That’s the first problem.

People say “agent” like it’s one category.

It isn’t.

A lot of teams are debating tooling, architecture, and workflow changes using a word they haven’t even defined.

---

# So they buy the wrong thing.

Some teams need:
- a coding agent
- a workflow agent
- orchestration
- a human-in-the-loop execution layer

Those are not interchangeable.

---

# If you don’t know which one you mean, the whole strategy gets fuzzy.

Bad tooling decisions.
Bad process design.
Bad expectations.

And then people blame “agents” when the real issue was sloppy framing.

---

# Before asking what agent stack to use, ask a better question:

What kind of agent problem are we actually trying to solve?

That question will save more time than the tool demo ever will.
