# The safest agent is the one with less power than you think it needs.

---

# Stop granting “full access” by default.

An agent usually does not need to send the email,
delete the file,
or make the irreversible change.

Reading is often enough.

---

# Good permission design is asymmetric.

Read broadly.
Write narrowly.
Delete almost never.

That one shift removes a huge amount of risk.

---

# Then add approval gates for destructive actions.

Yes, approvals create friction.

That is the point.

The right friction is cheaper than cleanup.

---

# Least privilege is not paranoia.

It is how you keep useful autonomy
without handing your system a loaded gun.

---

# If an action would hurt when done wrong,

make the agent ask.

Good agent design is not “maximum freedom.”
It is controlled leverage.
