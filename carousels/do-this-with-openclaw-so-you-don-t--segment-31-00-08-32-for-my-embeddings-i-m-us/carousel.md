# Serious agent stacks are multi-model by default.

---

# One model for everything is lazy architecture.

Planning, coding, search, embeddings, vision, and cheap background work
should not all hit the same model.

That is convenience masquerading as strategy.

---

# Use the expensive model where judgment matters.

Main planning.
Hard coding.
Security review.
High-stakes reasoning.

Use smaller or cheaper models where speed is enough.

---

# The goal is not “cheapest possible.”

The goal is:
use the best model for the decision,
and the cheapest model for the repetition.

That is how you keep quality high without burning budget.

---

# Routing beats brand loyalty.

Smart operators do not argue “Anthropic vs OpenAI vs open source.”

They build a routing layer.

Different jobs deserve different tools.

---

# If your agent stack feels expensive and mediocre,

it probably is not a model problem.

It is a model-assignment problem.
